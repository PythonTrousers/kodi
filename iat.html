Internet Archive Theater

v0.3.2 | Built for Kodi

Internet Archive Theater (IAT) is a dedicated video and audio plugin designed to interface with the Internet Archive. It provides a highly optimized, structured viewing experience for public domain and archived media directly within Kodi.

✨ Features & Additions

  • Continue Watching (Resume Data): Custom local JSON tracking remembers where you left off. Seamlessly integrates with Kodi's native UI to prompt you to resume or restart.
  • Granular Search Categories: Dedicated search routes for Movies, TV Shows, and Audio that intelligently target specific Internet Archive collections.
  • Favorite Collections Subfolders: Save custom Archive collections to a local favorites list, automatically organized into distinct media subfolders.
  • Binge-Watching / Auto-Play Queue: A threaded background tracker automatically queues up the next episode or track in a playlist sequence.
  • Smart Stream Selection: Extracts and displays resolution, file size, and source format before playback.

⚙️ Under-the-Hood Optimizations

  • Strict Media Filtering: Advanced regex and extension filtering automatically removes unplayable files, text documents, and junk data from results.
  • Smart Duration Parsing: Logic parses runtimes to automatically filter out promos, short clips, and trailers.
  • cURL Pipe Injection: Streaming URLs are injected with Keep-Alive parameters and User-Agent spoofing to bypass backend throttling and prevent dropouts.
  • DVD ISO Playback Optimization: Explicitly sets the MIME type to eliminate Kodi's internal network probe freezing on ISO files.

🔧 Troubleshooting: HTTP2 Stuttering

Newer versions of Kodi (v19 Matrix and above) utilize HTTP2 by default. This can cause severe stuttering, freezing, and playback tracking issues when streaming large files from backend servers like the Internet Archive.

If you are experiencing buffering loops or the addon fails to track your resume points, disabling HTTP2 in Kodi is the recommended fix:

  1. Locate your Kodi userdata folder (e.g., %APPDATA%\Kodi\userdata on Windows).
  2. Look for a file named advancedsettings.xml. If it does not exist, create a new text document and name it.
  3. Open the file in a text editor and add the following code:
<advancedsettings>
    <network>
        <disablehttp2>true</disablehttp2>
    </network>
</advancedsettings>

Save the file and restart Kodi. Your streams should now initialize via standard HTTP/1.1.
